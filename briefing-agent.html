The Daily Briefing Agent

An interactive guide for your book club's next project.

Project Goal & Core Concepts

This section provides a high-level overview of the project. The main goal is to build a simple agent that uses multiple tools to gather information (weather and news) and then uses a Large Language Model (LLM) to synthesize that information into a single, human-readable "daily briefing." This project is designed to solidify your understanding of several key agentic principles.

Tool Use & Workflow

Learn how to define and call distinct Python functions as "tools" and orchestrate them in a logical sequence.

Synthesis & State

Understand how an agent holds information (state) from its tools and uses an LLM to synthesize it into a cohesive output.

Step 1: User Trigger

The workflow starts when the user provides a simple prompt. The agent doesn't need to parse complex language for this project; it just needs to be triggered to start its process.

"Good morning, what's my daily briefing?"

Step 2: Information Gathering (Tools)

This is where the agent uses its tools. For this project, you will build two Python functions that act as tools to fetch external data. You can use real APIs or just return mock data to focus on the agent's logic. Select a tool below to see its required structure.

Function: get_weather(location: str) -> dict

Example Output (as a Python dictionary):

{'location': 'San Francisco, CA', 'temperature': '65F', 'condition': 'Cloudy'}

Function: get_top_headlines(topic: str, count: int = 3) -> dict

Example Output (as a Python dictionary):

{'topic': 'technology', 'headlines': ['New AI Model Released', 'Quantum Computing Breakthrough', 'Tech Stocks on the Rise']}

Step 3: Synthesis Phase

After gathering information from its tools, the agent combines the data into a single, comprehensive prompt. This prompt instructs the LLM to synthesize the structured data into a natural language paragraph.

Example Final Prompt:

You are a helpful assistant. Today is {current_date}.
Based on the following information, please write a short, friendly daily briefing:

Weather Data: {'location': 'San Francisco, CA', ...}
News Headlines: {'topic': 'technology', ...}

Combine this into a single, easy-to-read paragraph.

Implementation Plan

Here is your high-level plan. Follow these steps to build your agent. This interactive checklist will help you track your progress as you work through the project.

Create the `get_weather` and `get_top_headlines` Python functions. Test them to ensure they return data in the correct format.

Write a main function to orchestrate the workflow: call the tools and then pass their results to the synthesizer.

Create the `synthesize_briefing` function that constructs the final prompt and calls the LLM.

Stretch Goals

If you finish early and want an extra challenge, try tackling one of these optional goals. They introduce more advanced concepts like dynamic tool use and error handling.

  • Make the `location` and `topic` parameters dynamic based on user input.
  • Add a third tool, like a function that checks a calendar for the day's events.
  • Implement robust error handling for when an API tool fails to return data.
